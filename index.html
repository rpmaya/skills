01 ¿Qué son las Agent Skills?

Las Agent Skills son capacidades modulares que extienden la funcionalidad de Claude. Cada Skill empaqueta instrucciones, metadatos y recursos opcionales (scripts, plantillas, documentación de referencia) que Claude utiliza automáticamente cuando son relevantes para la tarea del usuario.

Analogía clave

Piensa en las Skills como una guía de onboarding para un nuevo empleado: no le das todo el manual el primer día, sino que le proporcionas lo necesario según la tarea. Claude carga la información por etapas, bajo demanda.

¿Por qué existen?

El problema que resuelven es la repetición de contexto. Sin Skills, cada conversación con un agente IA parte de cero. Las Skills permiten "enseñar una vez, usar siempre".

02 Arquitectura: Progressive Disclosure

El concepto arquitectónico central es la progressive disclosure (divulgación progresiva). La información se carga en tres niveles:

1

Metadatos

Al arrancar, Claude solo lee el frontmatter YAML de cada Skill instalada. Permite tener muchas Skills sin penalización en contexto.

~100 tokens / Skill
2

Instrucciones

Cuando la petición coincide con la description, Claude lee el SKILL.md completo y lo carga al contexto.

< 5.000 tokens
3

Recursos

Scripts, plantillas y referencias adicionales solo se cargan cuando se necesitan. Los scripts se ejecutan vía bash — solo entra el output al contexto.

On-demand

Frontmatter de ejemplo

---
name: pdf-processing
description: Extract text and tables from PDF files, fill forms, merge documents.
  Use when working with PDF files or when the user mentions PDFs, forms, or
  document extraction.
---

Estructura de archivos

pdf-skill/
├── SKILL.md          ← Instrucciones principales
├── FORMS.md          ← Guía de relleno de formularios
├── REFERENCE.md      ← Documentación detallada de API
└── scripts/
    └── fill_form.py  ← Script ejecutable (no consume contexto)

03 Creando tu primera Skill

Una Skill es simplemente un directorio que contiene un archivo SKILL.md:

mi-skill/
└── SKILL.md

Anatomía del SKILL.md

---
name: explain-code
description: Explains code with visual diagrams and analogies. Use when explaining
  how code works, teaching about a codebase, or when the user asks "how does this work?"
---

# Explain Code

When explaining code, always include:

1. **Start with an analogy**: Compare the code to something from everyday life
2. **Draw a diagram**: Use ASCII art to show the flow, structure, or relationships
3. **Walk through the code**: Explain step-by-step what happens
4. **Highlight a gotcha**: What's a common mistake or misconception?

Keep explanations conversational. For complex concepts, use multiple analogies.

Campos del frontmatter

CampoRequeridoRestricciones
nameSíMáx 64 chars · solo minúsculas/números/guiones · sin tags XML · sin palabras reservadas ("anthropic", "claude")
descriptionSíMáx 1024 chars · no vacío · sin tags XML
disable-model-invocationNotrue = solo el usuario puede invocar
user-invocableNofalse = solo Claude puede invocar automáticamente

Dónde almacenar las Skills

~/.claude/skills/ → Skills personales (todos los proyectos)

.claude/skills/ → Skills del proyecto (compartidas vía git)

mkdir -p ~/.claude/skills/mi-skill
cd ~/.claude/skills/mi-skill
touch SKILL.md

Se suben como archivos .zip desde Settings → Features.

cd mi-skill/
zip -r mi-skill.zip .
# Subir mi-skill.zip desde Claude.ai

Gestión mediante endpoints REST en /v1/skills.

curl -X POST https://api.anthropic.com/v1/skills \
  -H "x-api-key: $ANTHROPIC_API_KEY" \
  -H "anthropic-version: 2026-01-01" \
  -F "skill=@mi-skill.zip"

Probando la Skill

  1. Invocación automática: pregunta algo que coincida con la description.
  2. Invocación directa: usa el comando /explain-code (el name se convierte en slash command).

07 Troubleshooting

La Skill no se activa

  1. Revisa el campo description — debe ser específico y "pushy".
  2. Incluye triggers explícitos: "Use when the user mentions X, Y, or Z".
  3. Verifica que el name cumple las restricciones (solo minúsculas, números, guiones).
  4. Comprueba que la Skill está en el directorio correcto.

Errores de runtime

ErrorCausa probableSolución
Script fallaDependencia no disponibleVerificar paquetes pre-instalados
Output incorrectoInstrucciones ambiguasAñadir ejemplos al SKILL.md
Contexto desbordadoSKILL.md demasiado largoDividir en archivos, usar progressive disclosure
Skill no encontradaUbicación incorrectaVerificar path ~/.claude/skills/ o .claude/skills/
Limitaciones importantes
  • Sin sincronización cross-platform entre Claude.ai y API.
  • API: sin acceso a red, sin instalación de paquetes en runtime.
  • Claude.ai: acceso a red variable según configuración.
  • Claude Code: acceso completo a red y sistema de archivos.

08 Ejemplo práctico completo

Skill de revisión de código Python con checklist estructurado:

---
name: python-code-review
description: Reviews Python code for best practices, type hints, docstrings,
  and common pitfalls. Use when the user asks to review Python code, check
  code quality, find bugs, or improve their Python implementation. Also
  activate when the user pastes Python code and asks for feedback.
---

# Python Code Review

## Review checklist

When reviewing Python code, evaluate these areas in order:

### 1. Correctness
- Does the code do what it claims?
- Are there edge cases not handled?
- Are there off-by-one errors?

### 2. Type safety
- Are type hints present on function signatures?
- Are complex types properly annotated (e.g., `dict[str, list[int]]`)?

### 3. Documentation
- Do public functions have docstrings?
- Are parameters and return values documented?

### 4. Style and idioms
- Follows PEP 8?
- Uses Pythonic patterns (list comprehensions, context managers)?

### 5. Performance
- Are there unnecessary loops that could be vectorized?

## Output format

**Summary**: One paragraph overall assessment.
**Critical Issues**: Must-fix problems.
**Suggestions**: Improvements that would enhance quality.
**Positive Aspects**: What's done well (always include this).

09 Consideraciones de seguridad

Trátalo como instalar software

Las Skills son poderosas porque dan a Claude nuevas capacidades mediante instrucciones y código. El mismo nivel de precaución de instalar una librería aplica aquí.

  • Usa Skills solo de fuentes confiables (tú mismo o Anthropic oficial).
  • Audita exhaustivamente Skills de terceros: revisa SKILL.md, scripts y recursos.
  • Fuentes externas son peligrosas : Skills que hacen fetch de URLs pueden contener instrucciones maliciosas (prompt injection).

Vectores de riesgo

🔓

Exfiltración de datos

Scripts que envían información sensible a sistemas externos.

⚠️

Acceso no autorizado

Lectura o escritura en partes del sistema fuera del alcance esperado.

💉

Prompt injection

Instrucciones maliciosas inyectadas vía referencias remotas o contenido cargado.

11 Dreaming RESEARCH PREVIEW

Code with Claude 2026 · Nueva capa de memoria activa

Dreaming es un proceso programado que revisa las sesiones pasadas y los stores de memoria del agente, extrae patrones y cura recuerdos para que las Skills mejoren con el tiempo.

Mientras una Skill normal es estática (instrucciones fijas en SKILL.md), Dreaming permite consolidación pasiva: cuando el agente no está activo, Claude analiza sus interacciones y refina patrones reutilizables, similar a cómo el cerebro consolida memoria durante el sueño.

Cómo se integra con Skills

Dreaming] D --> P[Extracción de
patrones] P --> SK[Skill curada
con nuevos ejemplos] SK --> A[Agente mejorado
en próxima sesión]

Resultados publicados

Consideraciones de seguridad

Dreaming amplía la superficie de ataque: instrucciones maliciosas inyectadas en una sesión pueden persistir en memoria curada. Audita los stores de memoria tan cuidadosamente como el código de las Skills.

12 Outcomes PUBLIC BETA

Outcomes cambia el modelo de prompting: en lugar de instruir paso a paso, defines una rúbrica de éxito y el agente itera contra un grader independiente hasta cumplirla.

Patrón de uso con Skills

---
name: report-generator
description: Generates executive reports following company style guide.
outcome:
  rubric: |
    - Report contains executive summary (max 200 words)
    - All numeric claims have source citations
    - Tone is formal but accessible
    - Visualizations include alt-text for accessibility
  grader: claude-opus-4-7
  max_iterations: 5
---

# Report Generator

[Instrucciones procedimentales...]

Loop de iteración

flowchart LR P[Prompt + Skill] --> A[Agente
genera output] A --> G{Grader
independiente
evalúa rúbrica} G -->|❌ Fallos| F[Feedback específico] F --> A G -->|✅ Pasa| O[Output final]
Por qué es importante

El grader corre en su propio context window — no está contaminado por el razonamiento del agente. Esto reduce el riesgo de auto-engaño y mejora la fiabilidad de Skills sensibles a calidad (informes, código de producción, contenido legal).

Skills como bloques de orquestación

Cada sub-agente puede cargar Skills diferentes. Una Skill de orquestación describe el patrón de delegación: 

---
name: incident-investigation
description: Coordinates a multi-agent investigation for production incidents.
  Use when user reports an outage, anomaly, or unexplained system behavior.

  - name: log-detective
    skill: error-log-triage
  - name: metrics-analyst
    skill: grafana-query
shared_workspace: /tmp/incident-{{id}}/

Managed Agents NEW

Presentados en Code with Claude 2026 (mayo, Londres) y reforzados en mayo–junio, los Managed Agents son la materialización productizada de la orquestación multi-agente. El usuario describe una especificación; el agente elige tools, ejecuta extremo a extremo y entrega un PR listo para mergear.

Demo en vivo · Boris Cherny (Head of Claude Code)

En el keynote, Boris construyó en directo un dashboard de refunds con idempotencia, soporte multi-divisa y audit logging — mostrando varias sesiones de agente concurrentes con observabilidad en tiempo real y entrega de un PR mergeable.

Diferencia clave frente a un sub-agente clásico

Sub-agenteManaged Agent
Lo invoca el lead agent dentro de tu sesiónSe lanza como tarea independiente con webhook de finalización
Compartes tu ventana de contexto y toolsEspacio de trabajo aislado, tools seleccionadas por la spec
El control queda contigoEjecución autónoma con rúbrica (Outcomes) y memoria persistente (Dreaming)
Output: respuesta en chatOutput: PR / artefacto + registro auditable

Las Skills siguen siendo el lenguaje común: una Managed Agent carga las Skills relevantes para su dominio (refunds, migraciones, triaging) en lugar de re-empaquetar el conocimiento procedimental en cada spec.

16 Python SDK & portabilidad NEW

El Agent Skills SDK es la implementación de referencia en Python, diseñada para tratar Skills como unidades abstractas de expertise consumibles por cualquier framework de agentes.

Capacidades del SDK

  • Carga programática: descubrir Skills desde directorios, URLs o packages.
  • Validación: chequear que SKILL.md cumple el estándar.
  • Ejecución: invocar Skills desde código, con o sin agente.
  • Storage abstraction: locales, S3, GCS, repos git, registries privados.

Ejemplo de uso

from agent_skills import SkillRegistry, Skill

# Cargar Skills desde múltiples fuentes
registry = SkillRegistry()
registry.load_from_path("~/.claude/skills/")
registry.load_from_git("github.com/anthropics/skills")
registry.load_from_url("https://my-cdn/team-skills.zip")

# Descubrir Skills relevantes para una tarea
matches = registry.discover(
    task="Generar un dashboard de ventas mensual",
    top_k=3,
)
for skill in matches:
    print(f"{skill.name}: {skill.description[:80]}...")

# Ejecutar una Skill específica con tu agente preferido
result = registry.invoke(
    skill_name="dashboard-builder",
    inputs={"data_source": "sales.csv", "period": "monthly"},
    agent_backend="claude-opus-4-7",  # o "gpt-4", "gemini", etc.
)

Portabilidad cross-platform

Una Skill, múltiples agentes

El mismo SKILL.md que escribes para Claude funciona en Codex (OpenAI), Cursor, Gemini CLI y +20 plataformas adoptantes. Cero vendor lock-in a nivel de conocimiento procedimental.

17 Novedades junio 2026 JUST IN

Entre el cierre de Code with Claude 2026 y el inicio de junio, Anthropic ha publicado una tanda de mejoras incrementales que tocan de lleno el día a día de quien escribe Skills. Esta es la lista cerrada a 7 de junio de 2026.

♻️

/reload-skills

Claude Code v2.1.157 (31 mayo) introduce un comando que re-escanea los directorios de Skills sin reiniciar la sesión. Iterar sobre una Skill ya no obliga a abrir un Claude nuevo.

🪝

SessionStart → reloadSkills: true

Un hook de SessionStart que instale una Skill al arrancar puede ahora devolver { "reloadSkills": true }. Claude la recarga y queda disponible en la misma sesión.

📂

Auto-load de .claude/skills/

Los plugins ubicados en .claude/skills/ se cargan automáticamente, sin necesidad de marketplace. Además, claude plugin init hace el scaffolding inicial de un plugin nuevo.

🚫

disallowed-tools en frontmatter

Skills y commands pueden listar tools vetadas mientras están activas. Útil para sandbox: una Skill de revisión puede deshabilitar Bash aunque el usuario lo tenga permitido a nivel global.

🎛️

defaultEnabled: false

Un plugin puede declararse en plugin.json o en la entrada del marketplace como instalado pero desactivado por defecto. Reduce ruido en organizaciones con catálogos grandes.

🧠

Claude Opus 4.8 (28 mayo)

Mejora explícita en agentic skills, coding, razonamiento y trabajo práctico de conocimiento respecto a Opus 4.7. Cabecera API: claude-opus-4-8.

📅 Facturación separada del Agent SDK desde 15 de junio 2026

A partir del 15 de junio de 2026, el uso del Claude Agent SDK y de las Claude Code GitHub Actions se factura separadamente del Claude Code interactivo. Si distribuyes Skills empaquetadas para correr en CI/CD o vía SDK propio, revisa tu telemetría antes de la fecha de corte.

{
  "hooks": {
    "SessionStart": [
      {
        "type": "command",
        "command": "scripts/install-team-skill.sh && echo '{\"reloadSkills\": true}'"
      }
    ]
  }
}

Con reloadSkills: true, la Skill clonada por install-team-skill.sh entra en el índice antes del primer prompt del usuario — sin recargar Claude.
